# Tablas verificadas – Anuario 2023

Estos **100 archivos CSV** (56 cuadros del anuario, cuadros 3 a 58; varios cuadros generan 2 o 3 archivos) fueron extraídos del PDF del anuario "Seguro en Cifras 2023" y verificados (sumas vs TOTAL del documento y, donde aplica, cruce entre cuadros).

**Verificación del índice (recomendado al inicio):** Para comprobar que están cubiertos todos los cuadros del índice (págs. 4-6) y ver la **ubicación página → cuadro**, ejecutar: `python scripts/verificar_indice_anuario.py --year 2023`.  
**Consistencia reglas vs archivos:** `python scripts/verificar_consistencia_reglas_archivos.py --year 2023`.  
Las fórmulas y definiciones de los indicadores (Notas al final, págs. 137-139) se documentan en `docs/NOTAS_ANUARIO_PAGS_137_139.md`.

**Formato CSV (unificado):**
- **Separador único:** punto y coma (`;`) en todo el archivo (todas las columnas).
- **Comas en los datos:** La primera columna (Nombre Empresa, RAMO, etc.) puede contener comas (ej. "Seguros, C.A."). Esos valores se escriben entre comillas dobles para que la coma no se interprete como separador.
- **Por qué punto y coma y no coma:** Con coma como separador, cada "Seguros, C.A." partiría la columna en dos; además en muchos contextos la coma es el separador decimal. Con `;` hay un solo delimitador y las comas quedan solo dentro de los valores (y entre comillas si hace falta).

## Entender el flujo completo (ejemplo sencillo)

Ejemplo que asume el PDF en el mismo directorio que el script. Ejecutar desde la raíz o desde `docs/`:

```bash
# Opción 1: copiar el PDF a docs/ y ejecutar
cp data/raw/pdf/seguros-en-cifra-2023.pdf docs/
python docs/ejemplo_flujo_extraccion_anuario_pdf.py

# Opción 2: desde docs/ con el PDF ahí
cd docs && python ejemplo_flujo_extraccion_anuario_pdf.py
```

El script muestra en pantalla: 1) PDF usado, 2) tabla extraída, 3) CSV escrito, 4) lectura del CSV. Salida de ejemplo: `docs/ejemplo_salida.csv`.
